El proceso de login en una aplicación es el mecanismo de seguridad que permite verificar la identidad de un usuario mediante credenciales antes de concederle acceso al sistema. Es fundamental que ciertas rutas de la plataforma estén estrictamente protegidas porque manejan paneles administrativos, modificaciones de bases de datos o acciones críticas que podrían alterar o borrar solicitudes comunitarias. Sin esta restricción, cualquier atacante externo podría sabotear la base de datos de control social. Por motivos de seguridad y cumplimiento legal, bajo ninguna circunstancia debe exponerse públicamente información privada como contraseñas, documentos de identidad (cédulas), números telefónicos, direcciones residenciales, tokens de acceso o llaves de API que pongan en riesgo a la comunidad.
